Hur du sätter upp kopplingarna

Varje plattform kräver en utvecklar-app. Följ stegen nedan.

1

Skapa utvecklar-appar

Instagram/Meta: Gå till developers.facebook.com, skapa en app av typen "Business", lägg till "Instagram Graph API".
TikTok: Gå till developers.tiktok.com, skapa en app under "Login Kit".
LinkedIn: Gå till linkedin.com/developers, skapa en ny app kopplad till SSSB:s företagssida.

2

Lägg till redirect-URLer

I varje utvecklar-app, lägg till dessa som godkända redirect-URLer:

PlattformRedirect URI
Meta/Instagram
TikTok
LinkedIn
3

Lägg till miljövariabler i Vercel

Gå till Vercel-projektets Settings > Environment Variables och lägg till:

VariabelBeskrivning
META_APP_IDApp ID från Meta Developer Dashboard
META_APP_SECRETApp Secret från Meta Developer Dashboard
TIKTOK_CLIENT_KEYClient Key från TikTok Developer Portal
TIKTOK_CLIENT_SECRETClient Secret från TikTok Developer Portal
LINKEDIN_CLIENT_IDClient ID från LinkedIn Developer
LINKEDIN_CLIENT_SECRETClient Secret från LinkedIn Developer
SUPABASE_URLDin Supabase-URL (redan konfigurerad?)
SUPABASE_SERVICE_KEYSupabase service_role key (EJ anon key)
BASE_URLDin Vercel-URL, t.ex. https://sssb-studio.vercel.app
4

Skapa Supabase-tabell

Kör denna SQL i Supabase SQL Editor:

CREATE TABLE social_connections (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  platform TEXT UNIQUE NOT NULL,
  access_token TEXT NOT NULL,
  refresh_token TEXT DEFAULT '',
  platform_account_id TEXT DEFAULT '',
  account_name TEXT DEFAULT '',
  extra JSONB DEFAULT '{}',
  connected_at TIMESTAMPTZ DEFAULT now()
);

-- Row Level Security
ALTER TABLE social_connections ENABLE ROW LEVEL SECURITY;

-- Allow service role full access
CREATE POLICY "Service role access"
  ON social_connections FOR ALL
  USING (true) WITH CHECK (true);
5

Deploya och testa

Pusha koden till GitHub, vänta på att Vercel bygger om, och klicka sedan på "Koppla"-knapparna ovan. Du redirectas till varje plattform för att logga in.
